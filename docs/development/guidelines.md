# Development Guidelines

- Follow Clean Architecture principles: keep domain logic isolated.
- Write unit tests for all new features and policies.
- Use TypeScript for all code.
- Run lint and format scripts before committing.
- Document public APIs and important decisions.
- Prefer interfaces and abstract classes for extensibility.
- Use dependency injection for testability.

### Scripts

- `npm run build` — Build the project
- `npm run test` — Run tests
- `npm run lint` — Lint code
- `npm run format` — Format code
